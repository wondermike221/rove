## Summary

A dual-mode floating UI component (command palette + directory view) built with SolidJS, mounted in a shadow DOM, distributed as a vanilla JS library and userscript bundle. Command palette is the default mode. All shortcuts and display preferences are configurable at consumer and end-user levels. Fuzzy search via uFuzzy across the full node tree. State persisted to namespaced localStorage keys.

---

## Technical Context

**Language/Version**: TypeScript 5.8 / ES2020 targets

**Primary Dependencies**:
- SolidJS 1.9 (reactivity + rendering)
- solid-transition-group 0.3 (mode switch / overlay transitions)
- uFuzzy (fuzzy search, ~1.2kb gzip)
- Vite 6 + vite-plugin-solid + vite-plugin-dts (build)

**Storage**: localStorage — dot-notation namespace `{prefix}.{category}.{setting}`

**Testing**: Vitest + @solidjs/testing-library (unit + component tests)

**Target Platform**: Modern browser (Chrome 102+, Firefox 112+, Edge), ES2020+. Userscript environments (Tampermonkey, Violentmonkey).

**Project Type**: UI component library

**Performance Goals**: Fuzzy search results rendered within one animation frame (≤16ms) for ≤500 indexed nodes.

**Constraints**:
- Userscript IIFE bundle ≤100kb raw
- Zero style leakage to/from host page (shadow DOM)
- All non-global keyboard shortcuts strictly focus-scoped

**Scale/Scope**: Single component instance per init() call. Multiple instances supported (isolated by keyPrefix). Advisory tree limit: ~500 total nodes.

## Phase 1: Design

### Data Model

See `data-model.md` for full type definitions.

**Key entities**:
- `ConsumerConfig` — init-time config (tree, keyPrefix, defaults)
- `DirectoryNode` — `Record<string, DirectoryItem>` (key `meta` reserved)
- `DirectoryItem` — discriminated union: `directory | action | input | virtual`
- `InputItem` — sub-typed: `text | textarea | checkbox | select | select-multiple`
- `AppState` — root SolidJS store: `{ visible, mode, window, palette, nav, meta }`
- `SearchIndex` — flat parallel arrays: `haystack[]` + `SearchResult[]`
- `OverlayState` — `input | loading | error` discriminated union
- `MetaSettings` — resolved user preferences (consumer defaults → localStorage)

**State resolution order**: hardcoded defaults → ConsumerDefaults → localStorage overrides

### Interface Contracts

See `contracts/api.md` for full contract.

**Public surface**:
- `init(config: ConsumerConfig): ComponentInstance`
- `ComponentInstance`: `{ show, hide, toggle, destroy }`
- Userscript global: `window.__ROVE__ = { init }`

### Architecture Notes

**Mode switching**: Instant toggle. Mode state in `AppState.mode`. Dir view position/size persisted to localStorage; restored on each switch to dir view.

**Palette search flow**:
1. `AppState.palette.query` change → `fuzzy.search(index, query)` → `AppState.palette.results`
2. Results list re-renders with highlight ranges
3. `aria-live` status region announces result count

**Virtual node flow**:
1. Activation → `OverlayState = { type: 'loading', cancel }`
2. `load()` called; Esc dismisses overlay but fetch continues
3. On resolve → tree mutation → `SearchIndex` rebuild → palette results refresh
4. On reject → `OverlayState = { type: 'error', message }`

**Shortcut registry**:
- `registry.registerGlobal(shortcut, handler)` — listens on `document`
- `registry.registerScoped(shortcut, handler)` — listens on shadow host (focus-gated)
- `registry.updateShortcut(id, newShortcut)` — live rebinding (used by meta settings)

**Multi-instance**: Each `init()` call creates an isolated closure: own shadow root, own store instance, own registry. No shared global state.
